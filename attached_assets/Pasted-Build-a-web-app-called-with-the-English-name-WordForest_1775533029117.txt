Build a web app called "낱말의 숲" with the English name "WordForest".

Core concept:
This is an educational web app that starts from Korean words and helps users expand into deeper understanding, artistic imagination, and expressive creation. It is not just a dictionary search tool. It should help students and teachers explore words, expand related expressions, read texts deeply, and enrich dramatic expression for poetry, lyrics, and musical/drama activities.

Main Korean tagline:
낱말을 깊이 이해하고, 예술적 상상과 표현으로 확장해 가는 교육용 웹앱

Primary users:
- Elementary school students
- Elementary teachers
- Classroom activities involving poetry writing, lyric interpretation, educational musical, drama, and expressive language work

Main structure:
The app has 4 core sections:
1. 낱말 관찰
2. 낱말 확장
3. 깊이 읽기
4. 극적 표현

Overall learning flow:
낱말 관찰 → 낱말 확장 → 깊이 읽기 → 극적 표현

Product identity:
This app must not feel like a search engine for definitions.
It should feel like a creative language companion that helps users understand words and turn them into artistic expression.

Design and UX direction:
- Warm, calm, friendly, slightly poetic
- Forest-inspired visual identity
- Soft greens, natural tones, clean cards, rounded UI
- Easy enough for children, but useful for teachers
- Avoid clutter
- Make the interface feel like exploring and growing ideas in a forest
- The app should feel educational, imaginative, and creative rather than technical

Detailed feature requirements:

1. 낱말 관찰
Purpose:
Help users deeply observe one word at a time.

Key behavior:
- User enters a Korean word
- Show dictionary meaning, examples, and lexical info from the dictionary API
- Add simple, student-friendly explanation of the word using local placeholder logic
- Explain the feeling, tone, or possible mood of the word in accessible language
- Suggest where the word could be used, for example in poetry, a lyric, a character line, or a scene
- Allow saving interesting words

What this section should feel like:
A place to pause and really look at one word carefully.

Example outputs:
- Meaning of the word
- Example sentence
- Mood keywords
- “This word may fit a quiet / bright / trembling / lonely scene”
- “This word may work well in a poem about spring / memory / longing”

2. 낱말 확장
Purpose:
Help users broaden a theme or starting word into a wider set of expressive language materials.

Key behavior:
- User enters a topic or a word
- Suggest related words
- Suggest emotion words
- Suggest action verbs
- Suggest descriptive words or scene-related words
- Let users collect favorite words into a small word set or expression set
- Support creative activities such as poetry or scene-building
- Use local mock logic or curated demo relationships if necessary
- Do not use any external recommendation or language model API

What this section should feel like:
A place where one seed word grows branches.

Example:
Input: 비밀
Possible suggestions:
숨기다, 속삭이다, 떨리다, 조심스럽다, 들키다, 간직하다

This section should help students who do not know how to begin writing.

3. 깊이 읽기
Purpose:
Help users deeply interpret text such as poems, song lyrics, short passages, or musical numbers.

Key behavior:
- User pastes Korean text
- Extract important words
- Highlight meaningful, repeated, emotionally charged, or symbolic words
- Use dictionary-grounded interpretation where possible
- Provide student-friendly explanation of key words and overall emotional tone using local placeholder logic
- Optionally support two reading modes:
  - 학생용 해설
  - 교사용 해설
- Show the emotional atmosphere or expressive qualities of the text
- Do not call any external summarization, translation, or AI API

What this section should feel like:
A place for reading beyond the surface.

Important:
This is not just text summary.
It should help users understand:
- What kind of feeling the text creates
- Which words shape the mood
- Which repeated or symbolic words matter
- How the text might be expressed artistically

4. 극적 표현
Purpose:
Help users find richer dramatic and performative expression for scenes, emotions, and character moments.

Key behavior:
- User inputs a scene, situation, or emotional context
- Suggest action verbs
- Suggest emotional vocabulary
- Suggest attitudes, gestures, or delivery hints
- Suggest ways a line may be spoken
- Help students move from vague feelings to playable actions
- Implement suggestion logic with local mock data, predefined expressive categories, or internal rules
- Do not use any outside API except the dictionary API

What this section should feel like:
A place where language turns into performance.

Example input:
친구에게 미안하지만 쉽게 사과하지 못하는 장면

Possible output:
- Action verbs: 머뭇거리다, 피하다, 떨다, 멈칫하다, 숨죽이다
- Emotion words: 조심스럽다, 복잡하다, 불안하다, 미안하다
- Performance hints: 눈을 바로 보지 못한다, 말을 하려다 멈춘다, 숨을 짧게 들이쉰다

This section should support educational musical, drama, roleplay, and expressive speaking.
